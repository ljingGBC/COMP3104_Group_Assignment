
1. Make a group of 3 or 4 members only. No group of less than 3 or more
than 4 people will be allowed. After creating group start working on
your group assignment. (1 Point)

2. Create a git repository name COMP3104_Group_Assignment. Note
that only one member (Leader/Owner) from your group will create a
repository and other remaining member will clone the repository on their
local system. (1 Point)

3. The leader/owner of the repository will add other group member as
collaborator and assign required permission to work on the repository.
(1 Point)

4. All member will create their own branch on git repo having
STUDENTID as branch name. e.g. 1023756 (1 Point)

5. All group member will make atleast 10 commits and create 3 files on
their branch having appropriate commit message and merge all
branch changes to master branch.
Three file names for each group member will be in given below format
- Studentid_gb.txt (3 Points)
o This file will contain information about George Brown
college
- Studentid_devops.txt (3 Points)
o This file will contain information about COMP 3104
DevOps course
- Studentid_sdlc.txt (3 Points)
o This file will contain details about important of learning
DevOps engineer in software development cycle

6. Update your local branch from remote git repository and resolve any
merge conflict whenever required. (2 Point)

7. Group members should add and update Readme.md file to maintain
their repository details (1 Point)

Submission
Provide the GitHub group project link in assignment submission folder on
black board on or before deadline.
